SQL (Structured Query Language) - декларативна мова програмування для взаємодії користувача з реляційними базами даних.

1. База Даних
БД - це набір даних які зберігаються у систематичний та організований спосіб. Можуть бути збережені локально або на хмарі.
Структуровані або Неструктуровані дані. Керування через системи управління бази даних (СУБД).

2. DBMS (СУБД)
Database Management System(Система керування базами даних) - це програмний комплекс для створення, маніпуляції та управління БД.
Дозволяє великій кількості користувачів одночасно отримувати доступ до інформації, забезпучуючи послідовність та надійність і-ції, 
що зберігаються у БД.

!!!!!!!!!!!!!!СУБД на відміну від БД це програмний продукт. Є інтерфейс, таблиці, поле для коду для його результат.!!!!!!!!!!!!!!!!!!!!!!!

3. Структуровані дані:
- Висока організованість, ледка доступність
- Зазвичай у формі рядків (records) та стовпців (fields)
- Імена, номери карток, адреси
- Легко дослідити за допомогою мови SQL

4. Неструкуторвані дані
- Не можуть бути збережені в БД у форматі рядків та стовпців
- Зберігаються у Data Lakes, Data Warehouse, NoSQL - БД
- Частка - до 90% від усіх даних (електронні листи, відео, споціальні медіа, PDF-файли)

5. Реляційні БД - називаються БД які призначені для зберігання структурованих даних відповідно до реляційної моделі.
- Визначає зв'язки між таблицями БД.
- Для взаємодії з реляційною БД(RDB) використовується SQL.

6. SQL - використовується для створення, доступу та оновлення даних, збережених у реляційній БД
Підмножини мови SQL:
- DML (маніпулювання даними) - Select, Insert, Update, Delete
- DDL (визначення даних) - створення та модифікація даних, Create, Alter, Drop, Truncate
- DCL (контроль даних) - здійснюється адміністрування доступами користувачів до БД, Grant, Revoke
- TCL (контроль транзакцій) - Save point, Roll Back, Commit

7. SQL діалетки (Вендори)
- На основі SQL різні вендори розробляють свої діалетки
- Більшість функцій спільна для всіх діалектів :
                                                1. Microsoft SQL Server (T-SQL, SSMS - програмне забезпечення для взаємодії з БД)
                                                2. ORACLE (PL/SQL, SQL Developer)
                                                3. PostgreSQL (PL/PG SQL, pg Admin)
                                                4. MySQL (MySQL, Workbench)
                                                5. GoogleBigQuery(GoogleSQL, Google Cloud)

8. Процес
- Отримання доступ до бази даних
- Ознайомлення з Документацією
- Ознайомлення зі схемою, таблицями та зв'язками
- Писати маленькі запити і досліджувати дані
- Вирішити поставлені задачі
- Візуалізація, Дашборд, Звіти, Презентації

9. SQL DATA Types
- Кожен стовпець має назву та тип даних
- Забезпучує захист даних перевіркою типів даних
- Визначається, як SQL взаємодіятиме зі збереженими даними
- Основні категорії даних:
                            1. Character (Текстові дані)
                            2. Numeric (Числові)
                            3. Boolean 
                            4. Data/Time 
                            5. Other (Json etc.)
                            6. etc.
